**NavBar hamburger menu (per D-15 and UI-SPEC):**

Update `frontend/src/components/NavBarV2/NavBarV2.jsx`:

1. Add a `useState` for `mobileMenuOpen`
2. Add a hamburger button (3 horizontal lines icon, or use lucide-react `Menu` icon) visible only at `<= 768px`
3. At `<= 768px`, hide the center links div by default; show them in a vertical dropdown when hamburger is clicked
4. User name and logout button remain visible at all viewports
5. All nav links in hamburger menu have minimum 44px touch target height

Use Tailwind classes for responsive behavior:
- Hamburger button: `className="md:hidden"` (visible only below 768px)
- Center links: `className="hidden md:flex"` for desktop, shown via state toggle on mobile
- Mobile menu: absolute/fixed positioned dropdown below the nav bar

```jsx
import { useState } from 'react';
import { Menu, X } from 'lucide-react';

// Inside NavBarV2:
const [mobileMenuOpen, setMobileMenuOpen] = useState(false);

// Hamburger button (before or after logo):
<button
  className="md:hidden"
  onClick={() => setMobileMenuOpen(!mobileMenuOpen)}
  aria-label={mobileMenuOpen ? 'Close menu' : 'Open menu'}
  style={{ background: 'none', border: 'none', cursor: 'pointer', padding: '8px' }}
>
  {mobileMenuOpen ? <X size={24} /> : <Menu size={24} />}
</button>

// Desktop links (hidden on mobile):
<div className="hidden md:flex" style={centreLinksStyle}>
  {/* existing links */}
</div>

// Mobile menu (shown when mobileMenuOpen):
{mobileMenuOpen && (
  <div style={{
    position: 'absolute', top: '100%', left: 0, right: 0,
    background: 'var(--color-surface)',
    borderBottom: 'var(--border-width) solid var(--color-border)',
    padding: 'var(--space-2) 0',
    zIndex: 50,
  }}>
    {/* Same links as desktop, styled vertically with 44px min height each */}
    <Link to="/dashboard" style={{ display: 'block', padding: '12px 24px', minHeight: '44px', lineHeight: '20px' }}>Dashboard</Link>
    {/* ... all other links, including dynamic entity links */}
  </div>
)}
```

**Config-driven dashboard metrics (UX-06):**

Update Dashboard.jsx to display 3-5 key metrics. The dashboard currently shows student counts. Extend to show config-driven metrics:

1. Keep the existing student count metric
2. Add entity-based metrics using the entity registry API:
   - Fetch `getEntities()` via useQuery to get list of registered entities
   - For each entity with `auto_crud: true`, show a count metric card
3. Add module-specific metrics from the school choice domain:
   - Total students count (existing)
   - Graduated vs active student count
   - Plans generated count

The metrics grid should use the `Card` shadcn component (installed in Plan 05):

```jsx
import { Card, CardHeader, CardTitle, CardContent } from '@/components/ui/card';
import { useQuery } from '@tanstack/react-query';
import { getStudents } from '../../api/students';

// Inside Dashboard:
const studentsQuery = useQuery({ queryKey: ['students'], queryFn: getStudents });

const metrics = [
  { label: 'Total Students', value: studentsQuery.data?.length ?? '—' },
  { label: 'Active Students', value: studentsQuery.data?.filter(s => !s.graduated).length ?? '—' },
  { label: 'Graduated', value: studentsQuery.data?.filter(s => s.graduated).length ?? '—' },
];

// Render metrics grid:
<div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6">
  {metrics.map((m) => (
    <Card key={m.label}>
      <CardHeader>
        <CardTitle style={{ fontSize: 'var(--font-size-sm)', fontWeight: 'var(--font-weight-medium)', color: 'var(--color-text-secondary)' }}>
          {m.label}
        </CardTitle>
      </CardHeader>
      <CardContent>
        <p style={{ fontSize: 'var(--font-size-2xl)', fontWeight: 'var(--font-weight-medium)', color: 'var(--color-text-primary)' }}>
          {m.value}
        </p>
      </CardContent>
    </Card>
  ))}
</div>
```
