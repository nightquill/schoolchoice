## Spacing Scale

Declared values (multiples of 4 only). Source: existing `tokens.css` — carried forward verbatim.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding, tight inline spacing |
| sm | 8px | Chip padding, compact element gaps, list item vertical rhythm |
| md | 16px | Default form field padding, card body padding, input height contribution |
| lg | 24px | Section padding, card padding, tab panel top padding |
| xl | 32px | Layout column gaps, major section breaks within a page |
| 2xl | 48px | Page-level vertical spacing between top nav and content |
| 3xl | 64px | Maximum content width horizontal padding on large viewports |

Exceptions:
- Touch targets on mobile (375px viewport): minimum 44px height for all interactive controls (tab buttons, primary CTA, nav links in collapsed hamburger menu). This overrides the 32px/40px defaults where necessary.
- StudentProfile tab bar: tab buttons use 12px vertical padding (space-3) and 20px horizontal padding (space-5) to match the existing `Tabs.jsx` contract; do not increase on desktop.

## Typography

All sizes in px (rem equivalents in parentheses). Source: existing `tokens.css` — collapsed to 4 contracted roles.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px (1rem) | 400 (normal) | 1.5 | Default paragraph text, form field values, tab panel content |
| Label | 14px (0.875rem) | 500 (medium) | 1.25 | Form labels, table headers, nav links, chip text |
| Heading | 20px (1.25rem) | 500 (medium) | 1.25 | Page section headings, card titles, modal titles |
| Display | 24px (1.5rem) | 500 (medium) | 1.2 | Page-level headings (StudentProfile name, Dashboard section titles) |

Notes:
- 12px (xs) is permitted only for badges, chips, and timestamps — never for body copy or labels.
- Weights are strictly 400 and 500 only. Do not introduce 600 or 700 in Phase 3. (Source: D-12, "clean and minimal".)
- Font family: `system-ui, -apple-system, sans-serif` for all roles. No web font imports.

## Color

Source: existing `tokens.css` `#F8FAFC`/`#FFFFFF`/`#2563EB` palette — carried forward. Visual direction: clean, muted, Linear/Notion-inspired (source: CONTEXT.md D-12).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F8FAFC` | Page background (`--color-background`), tab panel backgrounds |
| Secondary (30%) | `#FFFFFF` | Cards, nav bar, modal surfaces, form containers, StudentProfile tab container |
| Accent (10%) | `#2563EB` | Reserved — see list below |
| Destructive | `#DC2626` | Destructive actions only — see list below |
| Success | `#16A34A` | ELIGIBLE badge, success toast, import confirmation indicator |
| Warning | `#D97706` | PENDING badge, data completeness warning, validation caution |
| Text primary | `#0F172A` | Headings, field values, active nav links |
| Text secondary | `#475569` | Placeholder text, inactive nav links, secondary metadata |
| Border | `#CBD5E1` | All borders — cards, inputs, tab bar underline (inactive), table rows |

Accent (`#2563EB`) reserved for:
1. Primary CTA button background (e.g., "Save Changes", "Add Student")
2. Active tab indicator (2px underline on active tab)
3. Active nav link text color
4. Focus ring on all interactive elements (`outline: 2px solid #2563EB`)
5. Hyperlinks within body text

Destructive (`#DC2626`) reserved for:
1. "Delete Student" button background (danger variant)
2. INELIGIBLE badge background
3. Form field error border and error message text
4. ErrorMessage component border and text

Do not use accent or destructive for decorative purposes. All other interactive states (hover, selected row) use `#F1F5F9` (Slate 100) as a background tint — no accent.

## Interaction Contracts

### StudentProfile Tab Navigation (UX-02)

- Tab bar renders at top of StudentProfile page, full width, border-bottom `1px solid #CBD5E1`
- Active tab: text color `#0F172A`, weight 500, 2px underline in `#2563EB`
- Inactive tab: text color `#475569`, weight 400, 2px underline transparent
- Tab switching: no page reload, no scroll reset, URL hash updates to `#personal` / `#grades` etc.
- Keyboard: Arrow keys navigate between tabs (existing `Tabs.jsx` contract preserved)
- Each tab panel: `role="tabpanel"`, `aria-labelledby` pointing to its tab button
- Mobile (≤ 768px): tab bar scrolls horizontally; `overflow-x: auto`, `white-space: nowrap`; no wrapping

### QueryBoundary (UX-03)

- Pending state: centered `LoadingSpinner` inside the boundary, 48px top padding above, 48px bottom padding below
- Error state: `ErrorMessage` component with the message from `error.message`; a "Try again" link (text button in accent color) that calls `refetch()`
- Success state: renders children as-is
- Boundary wraps individual data regions, not the full page — nav and page chrome remain visible during loading

### Auto-generated List View (PLAT-03)

- Header row: column names from entity schema `label` field, font-size 14px, weight 500, text color `#475569`
- Data rows: font-size 16px, weight 400, text color `#0F172A`, row hover background `#F1F5F9`
- Empty state: `EmptyState` component, message: "No [entity label plural] yet. Add the first one to get started."
- Row click: navigate to detail/edit form (`/entities/{name}/{id}`)
- No inline editing

### Mobile Responsive Layout (UX-05)

- Breakpoints: 375px (mobile), 768px (tablet), 1024px (desktop)
- Dashboard cards: single column at ≤ 768px, 2-column at > 768px and ≤ 1024px, 3-column at > 1024px
- StudentProfile: stacked layout at ≤ 768px (tab bar scrollable, tab panel full width)
- Nav bar: at ≤ 768px, center links collapse behind a hamburger button; right section (user name, logout) remains visible
- No horizontal overflow at any breakpoint on any page

## Accessibility Contract

- All interactive elements: visible focus ring `outline: 2px solid #2563EB; outline-offset: 2px`
- Color is never the sole carrier of meaning — badges include text labels (ELIGIBLE, INELIGIBLE, PENDING)
- All form inputs have a visible `<label>` with `htmlFor` pointing to the input id
- All buttons have either visible label text or `aria-label`
- Error messages use `role="alert"` so screen readers announce them without focus
- Modals: focus trap, Escape key to close, focus returns to trigger on close (existing `Modal.jsx` contract preserved in shadcn Dialog replacement)
- Images and icons: decorative icons use `aria-hidden="true"`; informational icons have `aria-label`
- Touch targets: minimum 44px height on all interactive elements at ≤ 768px viewport
