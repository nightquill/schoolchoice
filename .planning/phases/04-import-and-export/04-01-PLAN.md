```python
from __future__ import annotations

import csv
import io
from difflib import SequenceMatcher
from typing import Any

import openpyxl

from app.platform.yaml_loader import EntityConfig
from app.platform.crud_generator import build_pydantic_schema
```

**Functions to implement:**

1. `normalize(s: str) -> str` — lowercase, replace spaces/hyphens with underscores.

2. `auto_map_columns(file_columns: list[str], entity_fields: list[str]) -> dict[str, str | None]` — Use `SequenceMatcher.ratio()` with threshold 0.6. Matched fields removed from available pool. Returns `{file_col: entity_field_or_None}`.

3. `parse_csv(content: bytes, config: EntityConfig) -> dict` — Decode with `utf-8-sig` (strips BOM). Use `csv.DictReader`. Return `{"format": "csv", "sheets": None, "columns": [...], "preview_rows": [...first 5...], "all_rows": [...], "auto_mapping": {...}}`.

4. `parse_excel(content: bytes, config: EntityConfig) -> dict` — Use `openpyxl.load_workbook(io.BytesIO(content), read_only=True, data_only=True)`. Return `{"format": "excel", "sheets": wb.sheetnames, "selected_sheet": sheetnames[0] or None}`.

5. `parse_excel_sheet(content: bytes, sheet_name: str, config: EntityConfig) -> dict` — Load specific sheet, extract headers from row 0, preview from rows 1-3. Convert all cell values to str (None -> ""). Return `{"columns": [...], "preview_rows": [...], "all_rows": [...], "auto_mapping": {...}}`.

6. `validate_rows(rows: list[dict], mapping: dict, config: EntityConfig) -> tuple[list[dict], list[dict]]` — For each row, apply mapping (file_col -> entity_field), validate via `build_pydantic_schema(config, "ImportRow")`. Returns `(valid_rows, error_rows)` where error_rows have `{"row_index": i+2, "data": {...}, "error_reason": str(e)}`.

7. `find_duplicates(rows: list[dict], config: EntityConfig, db_session) -> list[dict]` — Determine key fields: use `config.key_fields` if non-empty, else first required string field. For each row, query DB for existing record matching key field values. Return list of `{"row_index": i, "data": row, "existing_id": str(existing.id), "key_field": field_name, "key_value": value}`.

8. `generate_error_csv(error_rows: list[dict], config: EntityConfig) -> str` — Build CSV string with entity field columns + `error_reason` column appended. Uses `csv.DictWriter` + `io.StringIO`.

All cell values from user uploads treated as untrusted. Use `str(value) if value is not None else ""` normalization. Never evaluate formulas (openpyxl `data_only=True` handles this).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User upload -> parse_csv/parse_excel | Untrusted file content crosses into parsing functions |
| User data -> validate_rows | Untrusted cell values pass through Pydantic validation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | parse_excel | mitigate | openpyxl `data_only=True` prevents formula evaluation; `read_only=True` prevents write-back attacks |
| T-04-02 | Tampering | parse_csv | mitigate | csv.DictReader handles quoting/escaping; `utf-8-sig` decoding strips BOM; all cell values cast to str |
| T-04-03 | DoS | parse_csv / parse_excel | mitigate | File size limit enforced at endpoint level (Plan 02); openpyxl `read_only=True` prevents full memory load |
| T-04-04 | Information Disclosure | validate_rows | accept | Validation errors include field name and value — acceptable since user uploaded the data themselves |
</threat_model>
