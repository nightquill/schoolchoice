Modify listQuery to include search/filter params:
```javascript
const listQuery = useQuery({
  queryKey: ['entities', name, debouncedSearch, filters],
  queryFn: () => {
    const params = {};
    if (debouncedSearch) params.q = debouncedSearch;
    const activeFilters = {};
    for (const [key, val] of Object.entries(filters)) {
      if (val == null || val === '') continue;
      if (typeof val === 'object' && val.from) activeFilters[`${key}__gte`] = val.from;
      if (typeof val === 'object' && val.to) activeFilters[`${key}__lte`] = val.to;
      if (typeof val === 'object' && val.min) activeFilters[`${key}__gte`] = val.min;
      if (typeof val === 'object' && val.max) activeFilters[`${key}__lte`] = val.max;
      if (typeof val === 'string') activeFilters[key] = val;
    }
    if (Object.keys(activeFilters).length > 0) params.filters = JSON.stringify(activeFilters);
    return getEntityList(schemaQuery.data?.table || name, params);
  },
  enabled: !!schemaQuery.data,
});
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input -> search query | Search text sent as q= param to backend; must not allow SQL injection |
| User filter -> filters JSON | Filter values serialized as JSON; backend parses with json.loads |
| Backend CSV -> client download | Exported data flows from server to client file download |
| Backend HTML -> client download | Plan HTML content flows from server to client file download |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-14 | Tampering | SearchFilterBar | mitigate | Search text sent as query param `q=`; backend uses SQLAlchemy parameterized ILIKE, never string interpolation |
| T-04-15 | Tampering | FilterControl | mitigate | Filter values serialized as JSON; backend validates field names via `hasattr(model_cls, key)` before applying |
| T-04-16 | Spoofing | ActionBar export | mitigate | Export uses Axios with auth interceptor (Bearer token); never uses `<a href>` which would bypass auth |
| T-04-17 | Information Disclosure | CSV export | accept | Authenticated users export data they already have read access to; consistent with existing CRUD access |
| T-04-18 | Spoofing | HTML export button | mitigate | exportPlanHTML uses Axios with auth interceptor; plan-export endpoint requires `Depends(get_current_user)` |
</threat_model>
