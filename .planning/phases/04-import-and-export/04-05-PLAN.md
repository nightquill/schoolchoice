**Test 1: CSV Import (DATA-01, DATA-03)**
1. Navigate to any entity list page (e.g., http://localhost:5173/entities/student or another registered entity)
2. Click "Import Data" button in the action bar
3. Verify the import wizard page loads at /entities/:name/import
4. Drop or browse for the test CSV file
5. Verify column mapping page shows:
   - Auto-mapped columns (name, email, age mapped to entity fields)
   - Preview data (first 3 rows visible under each column)
   - Green checkmark icons on auto-mapped columns
6. Click "Next" to validate
7. Verify validation summary shows valid/error counts with colored badges
8. Verify error rows show in a table with row number and reason
9. Click "Download error rows as CSV" — verify downloaded CSV has error_reason column
10. Click "Import N Valid Rows" — verify success toast appears
11. Verify redirect back to entity list shows newly imported data

**Test 2: Search (DATA-07)**
1. On the entity list page, type in the search bar
2. Verify results update after ~300ms pause
3. Verify "No results found" empty state when search matches nothing
4. Clear search — all rows return

**Test 3: Filter (DATA-08)**
1. If entity has enum fields, verify dropdown filter appears
2. Select a filter value — verify list updates
3. Click "Clear filters" — verify all rows return
4. If entity has numeric fields, verify min/max range inputs appear

**Test 4: CSV Export (DATA-04)**
1. Click Export dropdown
2. Click "Export filtered results" (with a search/filter active)
3. Verify CSV file downloads with correct filename pattern: entity-export-YYYY-MM-DD.csv
4. Open CSV — verify it contains only filtered rows
5. Click "Export all" — verify full dataset downloads

**Test 5: HTML Export (DATA-06)**
1. Navigate to a student plan page (e.g., /students/:id/plan)
2. Find and click "Export HTML" button (if wired to plan page)
3. Verify HTML file downloads and opens correctly in browser with all styling inline
4. If Export HTML is not yet on plan page, verify the API endpoint works via:
   ```
   curl -H "Authorization: Bearer <token>" http://localhost:8000/api/v1/entities/plan-export/<plan-id> -o test.html
   ```
   Then open test.html in browser.

**Test 6: Regression**
1. Navigate to /dashboard — verify it loads with student list
2. Click a student — verify profile loads with tabs
3. Navigate to /schools — verify school directory works
4. All pages should have no console errors

**Test 7: Mobile (375px viewport)**
