## Interaction Contracts

### Import Wizard Flow (DATA-01, DATA-02, DATA-03)

**Step 1: Upload**
- `FileUpload` component with `accept=".csv,.xlsx,.xls"`.
- Drag-and-drop zone: 2px dashed border, `#CBD5E1` default, `#2563EB` on drag hover.
- On file select: file name displayed, "Next" button (primary) enabled.
- If Excel file detected: proceed to Step 2 (Sheet Select). If CSV: skip to Step 3 (Column Mapping).

**Step 2: Sheet Select (Excel only, source: CONTEXT.md D-03)**
- List of sheet names from the workbook displayed as a Select dropdown.
- Default: first sheet selected.
- "Next" button proceeds to Column Mapping for the selected sheet.

**Step 3: Column Mapping (source: CONTEXT.md D-01, D-02, D-04)**
- Two-column layout inside a Card:
  - Left column: file column header name (Label size 14px, weight 500).
  - Right column: Select dropdown listing entity fields. Auto-mapped fields pre-selected.
  - Below each mapping row: data preview showing first 3 rows of that column (Body size 16px, text secondary color, monospace for values).
- Auto-mapping: columns with header names matching entity field names (case-insensitive, underscore/space normalized) are pre-selected. Auto-mapped rows show a green checkmark icon (lucide `Check`, 16px, `#16A34A`).
- "Not imported" section (source: D-04): unmapped file columns listed below a Separator. Each has a "Map to..." Select dropdown allowing manual assignment.
- "Next" button triggers backend validation.

**Step 4: Validation Preview (source: CONTEXT.md D-05, D-06, D-07)**
- `ValidationSummary` banner at top:
  - "[142] valid" Badge (success variant), "[8] errors" Badge (destructive variant), "[3] warnings" Badge (warning variant).
  - Font: Label size (14px), weight 500 for counts.
- Below banner: scrollable Table of error/warning rows.
  - Columns: Row #, Field, Value, Reason.
  - Error rows: left border 3px solid `#DC2626`, background `rgba(220, 38, 38, 0.06)`.
  - Warning rows: left border 3px solid `#D97706`, background `rgba(217, 119, 6, 0.06)`.
- Duplicate rows (source: D-07): shown with a "Duplicate" Badge (warning). Each duplicate row has inline radio buttons: Skip / Overwrite / Import as new. A batch checkbox "Apply choice to all duplicates" at the top of the duplicates section.
- "Download Errors" text button (destructive text color) triggers CSV download of failed rows with `error_reason` column appended.
- "Import [N] Valid Rows" primary CTA button. Disabled if 0 valid rows.
- "Back" secondary button returns to Column Mapping.

**Step 5: Confirm and Import**
- On CTA click: Progress bar displayed (shadcn Progress component).
- On completion: success toast via Sonner: "[N] [entity label plural] imported successfully."
- Redirect to entity list page showing newly imported data.
- On error: error toast: "Import failed. Please try again." with full error in console.

### Search and Filter (DATA-07, DATA-08)

**SearchFilterBar (source: CONTEXT.md D-08, D-09, D-10)**
- Position: fixed above entity list Table, below ActionBar. Always visible (not collapsible).
- Search input: full-width Input on the left with placeholder "Search [entity label plural]...". Lucide `Search` icon (16px) inside input as left adornment.
- Debounce: 300ms after last keystroke before query fires (source: D-09).
- Filter controls: rendered to the right of search input (desktop) or below on a second row (mobile, stacked).
- Filter type auto-detection (source: D-10):
  - Enum field -> Select dropdown with all enum values + "All" option.
  - Date field -> Popover with two date inputs (From / To).
  - Numeric field -> Two Inputs side by side (Min / Max) with 80px width each.
  - Text fields -> included in global search only, no dedicated filter control.
- Active filter state: filled filter controls show a small dot indicator or filled background to signal active filtering.
- "Clear filters" text button appears when any filter is active.
- Mobile (375px): search input full width, filters stack vertically below search.

### Entity List with Action Bar (DATA-04, DATA-11, DATA-12)

**ActionBar (source: CONTEXT.md D-12)**
- Horizontal bar at top of entity list page, below page title.
- Left side: "Import" Button (primary variant, lucide `Upload` icon 16px).
- Right side: "Export" DropdownMenu (outline variant, lucide `Download` icon 16px).
  - Menu items: "Export filtered results" (default), "Export all [entity label plural]".
  - Source: CONTEXT.md D-11 -- CSV export respects current search/filter.
- Background: `#FFFFFF`, bottom border `1px solid #CBD5E1`, padding `sm` (8px) vertical, `md` (16px) horizontal.
- Mobile: buttons shrink to icon-only with tooltip; search/filter stack below.

### CSV Export (DATA-04, source: CONTEXT.md D-11)

- Click "Export filtered results": immediate CSV download. No dialog.
- File name: `[entity-name]-export-[YYYY-MM-DD].csv`.
- Contains all visible fields for all rows matching current search/filter.
- "Export all" ignores active filters, exports full dataset.
- During export: button shows loading spinner (inline, replaces icon). No page navigation.

### HTML Report Export (DATA-06, source: CONTEXT.md D-14)

- Available from report/plan detail views (not the entity list).
- "Export HTML" Button (outline variant, lucide `FileDown` icon 16px).
- Generated file: self-contained HTML with all CSS and JS inline. Opens correctly in any browser without internet.
- Pattern: follows existing `plan_generator.py` Jinja2 + inline CSS/Chart.js approach.
- During export: button shows loading spinner. On completion: browser download triggers.

## Accessibility Contract

Carried forward from Phase 3, with Phase 4 additions:

- All interactive elements: visible focus ring `outline: 2px solid #2563EB; outline-offset: 2px`.
- Import wizard steps: use `aria-current="step"` on the active step indicator.
- Column mapping Select dropdowns: each has a visible `<label>` with the file column name.
- Validation error table: uses `role="alert"` on the summary banner so screen readers announce error counts.
- Duplicate resolution radio buttons: grouped in `<fieldset>` with `<legend>` describing the duplicate row.
- Search input: `role="searchbox"`, `aria-label="Search [entity label plural]"`.
- Filter controls: each labeled with the field name via `aria-label` or visible label.
- FileUpload zone: existing `aria-label` and `aria-busy` attributes preserved.
- Export dropdown: `aria-haspopup="menu"`, menu items have `role="menuitem"`.
- Progress bar during import: `role="progressbar"` with `aria-valuenow`, `aria-valuemin`, `aria-valuemax`.
- Touch targets: minimum 44px height on mobile for all buttons, dropdowns, and filter controls.
- Color is never the sole carrier of meaning -- error rows have text labels ("Error: ...") alongside red indicators; badges include text.
