```python
def load_task_yaml(path: Path) -> "TaskDefinition":
    from app.platform.task_engine import TaskDefinition
    with open(path) as f:
        raw = yaml.safe_load(f) or {}
    required = ["task_id", "name", "prompts", "output_schema", "jinja2_template"]
    for key in required:
        if key not in raw:
            raise ValueError(f"Task YAML missing required '{key}' field: {path}")
    prompts = raw["prompts"]
    if "system" not in prompts or "user" not in prompts:
        raise ValueError(f"Task YAML prompts must have 'system' and 'user': {path}")
    return TaskDefinition(
        task_id=raw["task_id"],
        name=raw["name"],
        description=raw.get("description", ""),
        data_slots=raw.get("data_slots", {}),
        system_prompt_template=prompts["system"],
        user_prompt_template=prompts["user"],
        output_schema=raw["output_schema"],
        jinja2_template=raw["jinja2_template"],
        max_tokens=raw.get("max_tokens", 4096),
        temperature=raw.get("temperature", 0.3),
        enable_chat=raw.get("enable_chat", True),
        max_context_tokens=raw.get("max_context_tokens", 60_000),
    )

def load_rules_yaml(path: Path) -> dict:
    with open(path) as f:
        raw = yaml.safe_load(f) or {}
    if "domain" not in raw:
        raise ValueError(f"Rules YAML missing required 'domain' field: {path}")
    return raw
```

The TaskEngine class must implement:
- `TASK_DIRS`: list of Path objects — `[Path("backend/app/platform/tasks"), Path("backend/app/modules/school_choice/tasks")]`
- `load_task(task_id: str) -> TaskDefinition`: search directories in reverse order (module overrides platform), call `load_task_yaml()` from yaml_loader, raise `ValueError` if not found
- `build_messages(task: TaskDefinition, entity_id: str, db: Session) -> list[dict[str, str]]`: resolve data slots, render Jinja2 templates, check token count via `litellm.token_counter()`, apply `_truncate_context()` if over `task.max_context_tokens`
- `execute_task(task_id: str, entity_id: str, db: Session) -> dict[str, Any]`: load task, build messages, call `call_ai()`, strip markdown fences, parse JSON, validate with `jsonschema.validate()`, return dict. Uses retry logic (MAX_RETRIES=2, total 3 attempts). On retry, append error message to messages list so AI can self-correct. On final failure, raise `HTTPException(502)`.
- `_resolve_data_slots(data_slots, entity_id, db) -> dict`: map slot types to data. "student" type loads Student from DB and returns dict with name, year_of_study, ielts_score, extra_curricular, awards. "matchmaker" type calls `run_matching(student_id, db)` and converts results to dicts.
- `_truncate_context(context, task) -> dict`: copy context, reduce matchmaker to top 5 by final_score, truncate rationales to first sentence, remove shap_explanation dicts.
- `validate_all_task_yamls() -> list[str]`: classmethod that scans all TASK_DIRS, parses each YAML, returns list of errors. Called at startup for early failure detection (per RESEARCH.md Pitfall 4).

PII blocklist check in `build_messages()`: after rendering prompts, scan for forbidden field names `["medical_notes", "disciplinary_record", "sibling_data", "ssn", "dob"]`. If found, raise `ValueError("PII scope violation")`.
