**RecommendationEngine class:**

```python
class RecommendationEngine:
    def __init__(self, rules_config: dict):
        """
        rules_config is the parsed YAML dict from load_rules_yaml().
        Expected keys: domain, confidence_thresholds, eligibility_rules,
                       scoring_weights, ml_model (optional).
        """
```

Methods:
- `evaluate_eligibility(source_data: dict, target: dict) -> tuple[bool, list[str]]`: iterate `self.rules_config["eligibility_rules"]`, for each rule of type "comparison": extract source field value, extract threshold (from target via threshold_field or from threshold_value), apply operator comparison. If `skip_when_null` and source value is None, skip. For type "custom", call registered hook function. Return `(all_passed, failing_criteria_list)`.
- `compute_weighted_score(source_data: dict, target: dict, scoring_hooks: dict[str, Callable]) -> tuple[float, dict]`: read `self.rules_config["scoring_weights"]`, for each weight key call the corresponding function from `scoring_hooks` dict (e.g. `scoring_hooks["academic_fit"](source_data, target)`). Multiply each component score (0.0-1.0) by its weight. Return `(total_weighted_score, component_scores_dict)`.
- `blend_ml_score(weighted_score: float, ml_probability: float | None) -> float`: if ml_model config exists and `ml_probability` is not None, blend: `final = (1 - blend_weight) * weighted_score + blend_weight * ml_probability`. Otherwise return weighted_score. `blend_weight` from `self.rules_config.get("ml_model", {}).get("blend_weight", 0.4)`.
- `compute_confidence_tier(data_completeness: float) -> tuple[str, str]`: read thresholds from `self.rules_config.get("confidence_thresholds", {"high": 0.7, "medium": 0.4})`. Return `("HIGH", "All eligibility data complete.")` if > high, `("MEDIUM", "Some data missing. Confidence reflects available data only.")` if > medium, else `("LOW", "Significant data missing. Confidence reflects available data only.")`. Per D-15, D-16.
- `is_shap_enabled() -> bool`: return `self.rules_config.get("ml_model", {}).get("shap_enabled", False)`. Per D-17.
- `run_recommendations(source_data: dict, targets: list[dict], scoring_hooks: dict, completeness_fn: Callable, ml_score_fn: Callable | None = None, shap_fn: Callable | None = None) -> list[RecommendationResult]`: orchestrator that chains eligibility -> scoring -> ML blend -> confidence for each target. Sort by final_score descending. This is the generic equivalent of matchmaker_v2.run_matching().

The engine does NOT import from matchmaker_v2.py directly. It is domain-agnostic. The school choice domain will pass its specific hook functions (scoring components, completeness, ML, SHAP) when calling `run_recommendations()`.

Comparison operators supported: `>=`, `<=`, `>`, `<`, `==`, `in` (for list membership). Raise `ValueError` for unknown operators.
