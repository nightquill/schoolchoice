**Endpoint 1: GET /consultant/tasks/{task_id}/stream**

Use GET (not POST) because EventSource is GET-only (RESEARCH.md A1 resolution). The operation is idempotent.

```python
@router.get("/tasks/{task_id}/stream")
async def stream_consultant_task(
    task_id: str,
    entity_id: str = Query(...),
    db: Session = Depends(get_db),
    current_user: User = Depends(get_current_user),
) -> StreamingResponse:
```

Implementation:
- Instantiate `TaskEngine()`
- Call `engine.load_task(task_id)` — raise HTTPException(404) if ValueError
- Call `engine.build_messages(task_def, entity_id, db)` — raise HTTPException(400) if ValueError (PII violation, entity not found)
- Return `StreamingResponse(call_ai_stream(messages, max_tokens=task_def.max_tokens, temperature=task_def.temperature), media_type="text/event-stream", headers={"Cache-Control": "no-cache", "X-Accel-Buffering": "no"})`

**Endpoint 2: POST /consultant/tasks/{task_id}/save**

```python
@router.post("/tasks/{task_id}/save", status_code=status.HTTP_200_OK)
def save_consultant_task(
    task_id: str,
    body: ConsultantSaveRequest,
    db: Session = Depends(get_db),
    current_user: User = Depends(get_current_user),
):
```

Implementation:
- Parse `body.ai_output_json` as JSON — raise HTTPException(400) if invalid
- Load task definition via `TaskEngine().load_task(task_id)`
- Validate against task's output_schema via `jsonschema.validate()`
- Validate via Pydantic `ConsultantPlanOutput(**parsed_json)` for semantic checks
- Apply confidence guardrail: for each recommended_school, compute `compute_data_completeness()` via matchmaker_v2, compute tier via RecommendationEngine, downgrade AI tier if higher than code-computed tier
- Load or create AcademicPlan DB row for the entity_id (existing student plan pattern from plan.py)
- Update: `html_content` (render Jinja2 template with validated data), `recommended_schools` (JSON), `action_items` (JSON), increment `version`
- Commit and return `ConsultantTaskResponse`
