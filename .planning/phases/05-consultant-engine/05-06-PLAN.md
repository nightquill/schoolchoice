<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Complete consultant engine with:
1. YAML-driven task definitions (academic_plan.yaml)
2. SSE streaming plan generation via call_ai_stream()
3. TaskEngine with data slot resolution, prompt rendering, JSON validation
4. RecommendationEngine with YAML rules, confidence tiers, SHAP support
5. ConsultantTaskPage with SSEStreamDisplay and ConfidenceBadge
6. Jinja2 template hierarchy (base + professional/modern/minimal)
7. Chat modification and template switching preserved from existing system
  </what-built>
  <how-to-verify>
**Pre-requisite:** Ensure AI_API_KEY and AI_PROVIDER are set in the .env file. The backend must be running on port 8000 and frontend on port 5173 (or configured port).

**Step 1: Navigate to the consultant page**
1. Open http://localhost:5173/dashboard
2. Click on any student with grade data
3. Navigate to the student's consultant page (/students/{id}/consultant)
4. Expected: Page loads with toolbar showing student name, "Generate Plan" button, template selector, and empty state message

**Step 2: Generate a plan with SSE streaming**
1. Click "Generate Plan"
2. Expected: Button changes to "Generating..." with spinner; SSEStreamDisplay appears with "Generating plan..." label
3. Expected: Tokens appear progressively (within 1-2 seconds of first token)
4. Expected: After stream completes, rendered plan appears in the main area
5. Expected: Success toast "Plan ready - view it here."

**Step 3: Verify confidence badges**
1. Look at the recommended schools section in the rendered plan
2. Expected: Each school has a color-coded badge (HIGH=green, MEDIUM=yellow, LOW=red)
3. Hover over a badge
4. Expected: Popover tooltip shows either "All eligibility data complete." or "Missing: [fields]..."

**Step 4: Test stop generation**
1. Click "Generate Plan" again
2. While streaming, click "Stop Generation"
3. Expected: Stream stops immediately, no partial plan saved, UI returns to previous state

**Step 5: Test chat modification**
1. With a generated plan visible, type "Change the first school's rationale to emphasize language programs" in the chat panel
2. Expected: AI responds with a JSON patch, plan updates

**Step 6: Test template switching**
1. Click on "Modern" in the template selector
2. Expected: Plan re-renders with teal color scheme
3. Click on "Minimal"
4. Expected: Plan re-renders with minimal gray theme

**Step 7: Test HTML export**
1. Click "Export HTML"
2. Expected: HTML file downloads, opens in browser showing the complete styled plan

**Step 8: Mobile responsiveness**
1. Resize browser to < 768px width
2. Expected: Single column layout, chat panel collapsible below plan

**Step 9: Regression check**
1. Navigate to /students/{id}/plan (old plan route)
2. Expected: Old AcademicPlan page still works
3. Navigate to /dashboard
4. Expected: Dashboard loads correctly
  </how-to-verify>
