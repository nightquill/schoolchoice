## Spacing Scale

All values are multiples of 4. Sourced from `src/utils/tokens.css`.

| Token | CSS Variable | Value | Usage |
|-------|-------------|-------|-------|
| xs | `--space-1` | 4px | Icon gaps, tight inline padding |
| sm | `--space-2` | 8px | Badge padding, compact element gaps |
| md | `--space-3` | 12px | Chat message padding, dense section gaps |
| base | `--space-4` | 16px | Card padding, default form element spacing |
| lg | `--space-6` | 24px | Section padding, toolbar padding |
| xl | `--space-8` | 32px | Major section breaks, page-level top padding |
| 2xl | `--space-12` | 48px | Page-level vertical breathing room |

**Exceptions:**
- 12px (`--space-3`): sourced from existing `tokens.css` and used in existing `AcademicPlan.jsx` chat message padding — established pattern, not a new value. Retained as a documented non-standard step between 8px and 16px.
- Confidence badge touch target minimum: 44px height on mobile (WCAG 2.5.5)
- Chat panel fixed width on desktop: 360px (matches existing `AcademicPlan.jsx` chat column)
- SSE streaming text area minimum height: 200px to prevent layout jump during generation

## Typography

All values are sourced from `src/utils/tokens.css`. Use exactly these four roles — no additional sizes. Exactly two font weights are used: 400 (normal) and 700 (bold).

| Role | CSS Variable | Size | Weight | Line Height | Usage |
|------|-------------|------|--------|-------------|-------|
| Body | `--font-size-md` / `--font-weight-normal` | 16px (1rem) | 400 | 1.5 (`--line-height-normal`) | Chat messages, section descriptions, form help text |
| Label | `--font-size-sm` / `--font-weight-normal` | 14px (0.875rem) | 400 | 1.5 | Confidence badge text, section labels, toolbar metadata, chat input placeholder — differentiated from Body by smaller size and UPPERCASE case transform where needed |
| Heading | `--font-size-xl` / `--font-weight-bold` | 20px (1.25rem) | 700 | 1.25 (`--line-height-tight`) | Panel headers (AI Assistant, Fit Analysis), task page section titles |
| Display | `--font-size-2xl` / `--font-weight-bold` | 24px (1.5rem) | 700 | 1.25 | Page-level headings (Recommendations for [Name]) |

**Weight summary:** 400 (`--font-weight-normal`) and 700 (`--font-weight-bold`) only. No 500 / medium weight is introduced.

**Label differentiation without weight:** Use `font-size: 14px` + `letter-spacing: 0.05em` + `text-transform: uppercase` for tier labels (LOW / MEDIUM / HIGH) and metadata labels where visual separation from body copy is needed.

**Streaming text rendering:** Use `--font-size-md` / `--font-weight-normal` / line-height 1.6 inside the SSE streaming output container (extra breathing room compensates for token-by-token append). `font-family: var(--font-family-base)`.

## Component Inventory

New components required by Phase 5. All extend existing patterns — no new design language.

### 1. ConfidenceBadge

**Purpose:** Displays LOW / MEDIUM / HIGH tier next to a school recommendation. Includes tooltip with missing-field list.

**Anatomy:**
- Outer: `<span>` or shadcn `<Badge>` variant
- Text: tier label in uppercase (`LOW`, `MEDIUM`, `HIGH`)
- Tooltip: shadcn `<Popover>` triggered on hover/focus; lists missing field names as a comma-separated sentence

**States:**
- Default: colored badge (see color table above)
- Focused: `--color-primary` ring (2px offset)
- Tooltip visible: popover below badge, `--color-surface` background, 1px `--color-border` border, `--font-size-sm` text

**Size:** Minimum 44px height on mobile (touch target). Desktop: inline height ~22px.

**Accessibility:** `aria-label="Confidence: HIGH — all eligibility fields complete"` (or equivalent with missing field list for LOW/MEDIUM). Tooltip must be keyboard-reachable via focus.

---

### 2. SSEStreamDisplay

**Purpose:** Renders the live AI output stream during plan generation. Replaces the existing polling-based loading spinner with token-by-token text append.

**Anatomy:**
- Container: `<div>` with `overflow-y: auto`, `min-height: 200px`, `max-height: calc(100vh - 220px)` on the plan page
- Streaming text: plain `<p>` tags appended as tokens accumulate; `--font-size-md`, line-height 1.6
- Progress indicator: single-line `<p>` in `--color-text-secondary` at top of container: "Generating plan…"
- Cursor: blinking block cursor after the last token (CSS `::after` pseudo with `animation: blink 1s step-end infinite`)
- After done: cursor disappears; container scrolls to top

**States:**
- Idle: hidden (zero height, no render)
- Streaming: visible; auto-scroll to bottom on each token append
- Error: `border: 1px solid var(--color-error)` on container; error copy replaces streaming text; "Try Again" button (primary) below

**Accessibility:** `role="status"` `aria-live="polite"` on the container. Screen readers announce new content without interrupting.

---

### 3. ConsultantTaskPage (new route)

**Purpose:** Full-page view for triggering a YAML-defined consultant task and displaying the streaming output. This is the generalized replacement for `AcademicPlan.jsx` — school choice plan generation is the first task migrated to this page.

**Primary focal point:** The SSEStreamDisplay / rendered plan iframe in the left column is the primary visual anchor of this page. Before generation begins, the "Generate Plan" button in the toolbar is the CTA anchor that draws the user's eye. During and after generation, the streaming output becomes the focal point — it occupies the majority of the viewport (flex: 1) and receives auto-scroll attention as tokens arrive.

**Layout (desktop, ≥768px):**
- Left column (flex: 1, min-width: 0): SSEStreamDisplay or rendered plan iframe
- Right column (360px fixed): ChatPanel (existing from `AcademicPlan.jsx`)
- Both columns separated by `1px solid var(--color-border)`
- Top: toolbar (matches existing `AcademicPlan.jsx` toolbar pattern)

**Layout (mobile, <768px):**
- Single column
- Toolbar stacks vertically, buttons full-width
- Chat panel renders below the plan output, full width
- Chat panel collapsible via a "Show AI Chat" toggle button

**Toolbar contents (left to right):**
1. Student/entity name (`--font-size-lg`, `--font-weight-bold`)
2. Plan version if available (`--font-size-xs`, `--color-text-secondary`)
3. Template selector (existing `TemplateSelector` component, unchanged)
4. "Generate Plan" primary button (accent fill)
5. "Export HTML" ghost button with `FileDown` icon (16px)
6. Generated-at timestamp (`--font-size-xs`, `--color-text-secondary`)

**New prop:** `confidenceTier: 'LOW' | 'MEDIUM' | 'HIGH'`, `missingFields: string[]`

## Interaction Contracts

### SSE Streaming — "Generate Plan" flow

1. User clicks "Generate Plan" → button enters loading state (spinner, disabled, label "Generating…")
2. `EventSource` opens to `/api/v1/consultant/tasks/{task_id}/stream?entity_id={id}`
3. SSEStreamDisplay becomes visible; progress copy "Generating plan…" appears
4. Tokens append progressively; container auto-scrolls to bottom
5. `done` event received: cursor disappears; backend DB save triggered; plan re-fetched via existing `loadPlan()` pattern; button returns to idle state; success toast "Plan ready — view it here."
6. `error` event or `onerror`: container shows error state; "Generation was interrupted. Please try again." copy; button returns to idle

**Cancel during streaming:** "Stop Generation" button (secondary variant) visible while streaming. Click closes `EventSource` immediately; button returns to idle; SSEStreamDisplay resets to idle state. No partial plan is saved.

### Confidence Badge — hover/focus

1. Badge renders inline next to school name at all times (no hover-to-reveal)
2. Tooltip (shadcn `<Popover>`) appears on hover (200ms delay) or focus (immediate)
3. HIGH tooltip: "All eligibility data complete."
4. MEDIUM/LOW tooltip: "Missing: [field1], [field2]. Confidence reflects available data only."
5. Tooltip dismisses on mouse-leave or blur

### Template Selector

No changes from existing `TemplateSelector` interaction. Keep existing behavior intact.

### TipTap Section Editor

No changes from existing `PlanSectionEditor` interaction. Keep existing behavior intact.

## Accessibility Requirements

- All interactive elements have visible focus ring using `--color-primary` (`box-shadow: 0 0 0 2px #2563EB`)
- `ConfidenceBadge` tooltip accessible via keyboard (focus triggers popover)
- SSEStreamDisplay uses `role="status"` + `aria-live="polite"`
- Plan iframe preserves existing `title` and `aria-label` attributes
- Chat panel preserves existing `aria-label="Chat message input"` and `aria-label="Send message"` on textarea and button
- Mobile touch targets: minimum 44px height for all buttons and badges (WCAG 2.5.5)
