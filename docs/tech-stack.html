Matching Pipeline

POST /api/v1/students/{id}/match — triggered from Target Schools page or profile

1
Data Assembly Backend
build_student_data()
app/services/student_data_builder.py
Loads student ORM + grades + IELTS + activities + awards. Computes best5_aggregate via hkdse_service (4 compulsory + best elective, ApL excluded). Builds canonical dict for scoring.
2
Eligibility Filter Backend
run_eligibility_filter()
app/modules/school_choice/services/matchmaker_v2.py
Hard pass/fail: (1) best5 ≥ school minimum entry score, (2) each required subject meets min grade, (3) IELTS ≥ school requirement. Failing criteria returned as human-readable strings.
3
Scoring Backend
JUPAS Parametric Path (primary)
app/modules/school_choice/services/jupas_scorer.py
370+ real JUPAS programmes. Per-programme published formula: best_n subjects, per-subject weights, bonus weights. Grade → points via scale. Piecewise probability mapping (LQ→0.25, Med→0.50, UQ→0.75, exponential tails). Final: 0.8×admission_prob + 0.2×interest_alignment.
Heuristic Fallback Path
app/modules/school_choice/services/matchmaker_v2.py
For schools without JUPAS data. Weighted score: academic fit (50%) + subject alignment (20%) + language fit (15%) + interest alignment (15%). Optional XGBoost ML blend: 0.6×heuristic + 0.4×ml_prob.
Data completeness indicator (BUG-02): Computes fraction of expected grades present. compulsory_present/4 × 0.7 + min(elective_present/2, 1.0) × 0.3. Returned with every MatchResult.
4
Persist & Return Database
Upsert StudentSchoolTarget
Scores saved to student_school_targets table. Ranked by final_score descending, eligible first. At-risk flag set when score < programme lower quartile. SHAP explanations (top 3 features) stored as JSON.

Scoring Formulas

HKDSE Grade Scale

Grade5**5*54321U/X
Points76543210

CSD (Citizenship): Attained=1pt, Attained with Distinction=2pts. Best-5 = 4 compulsory (CHLA+ENGL+MATH+CSD) + best elective. ApL excluded.

JUPAS Parametric Scoring (primary path)

StepCalculation
1. Grade → PointsPer-programme grade scale (e.g. jupas_2025_enhanced). Each grade maps to a float.
2. Subject WeightingPoints × subject_weight (e.g. ENGL × 1.5, MATH × 1.5). Select best_n by weighted value.
3. BonusBest remaining subject × bonus_weight (e.g. 0.5 for 6th subject)
4. ProbabilityPiecewise interpolation: LQ→0.25, Median→0.50, UQ→0.75. Exponential tails above/below.
5. Final Score0.8 × admission_probability + 0.2 × interest_alignment

Heuristic Scoring (fallback path)

ComponentWeightCalculation
Academic fit50%best5 / avg_admitted_score (fallback: min×1.2, or 35 max)
Subject alignment20%Matched elective keywords / total electives (neutral 0.3)
Language fit15%IELTS ratio or ENGL grade/7 (neutral 0.65)
Interest alignment15%Keyword overlap (activities+awards vs programs) with synonym expansion

Multi-Tenancy & Auth

Authentication
JWT (HS256) with claims: sub (user UUID), org_id (organisation UUID), exp, iat.
Passwords hashed with bcrypt. Token stored in localStorage.
Axios interceptor: auto-inject Authorization: Bearer {token}. 401 → clear token → redirect /login.
Rate limits: register (10/min), login (15/min), plan chat (20/plan/24h).
Organisation Isolation
User → OrganisationMembership → Organisation
JWT carries org_id. get_current_user() attaches active_organisation_id to User object.
All student queries filtered: Student.organisation_id == user.active_organisation_id.
Roles: owner, admin, member. Enforced at service layer.

Security & Privacy

Authentication & Authorization
JWT + bcrypt. Role-based access (counsellor, admin). Rate limiting on auth + AI endpoints. Secrets in .env (gitignored).
PII Protection (PDPO)
PII blocklist scanned before AI calls: medical_notes, disciplinary_record, sibling_data, ssn, dob. Student names anonymized for third-party AI. Consent tracking per student.
Output Sanitization
AI output: control chars stripped, <script> / onerror= / javascript: flagged. HTML plan values escaped via _esc(). Plan sections sanitized with nh3.

Architectural Decisions

#DecisionRationale
D-01Organisation-based multi-tenancy (not row-level)Schools are separate institutions; org_id on JWT keeps queries simple
D-02pnpm monorepo (apps + packages)Shared UI library, single lockfile, workspace-level scripts
D-07Manifest-based module discoveryDomain modules self-declare via config.yaml; platform auto-registers
D-08Plugin directory structureEach module owns: models, services, routes, entities, templates, tasks
D-09JUPAS parametric scoring over MLPublished formulas are deterministic and auditable; ML is optional fallback
D-10JSONB for flexible fieldsgrades, interests, ielts_score, etc. evolve without schema migrations
D-14@media print CSS on plan templatesPlans print cleanly without separate PDF generation
D-15Cohort reports (target distribution, risk, subject performance)Counsellors need class-level analytics, not just per-student
D-19Annual data refresh with diff previewAdmin can preview CSV changes before committing to DB
D-20PDPO consent managementHong Kong privacy law requires explicit consent tracking
D-21LLM reporter persona with grounding rulesAI generates rationales grounded in actual student data, not hallucinated
