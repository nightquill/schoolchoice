SchoolChoice — UX Walkthrough

How school counsellors actually use the platform, step by step

Hong Kong secondary school context • HKDSE & JUPAS system • As of 2026-05-12

Who Uses This

MC
Ms. Chan — School Counsellor
St Paul's College, Hong Kong • Form 5–6 students • HKDSE university guidance
Ms. Chan manages 150 students across 5 classes (5A–5E). She tracks their HKDSE mock/trial/official grades, helps them pick target universities via JUPAS, generates personalised academic plans, and monitors cohort-level analytics to spot at-risk students before it's too late. She uses SchoolChoice daily during the academic year.
Context: In Hong Kong, Form 5–6 students sit the HKDSE exam (4 compulsory + 2–4 electives). Universities admit via JUPAS using a "best 5" aggregate score. Each programme has its own formula, weights, and admission statistics. Counsellors help students pick realistic targets and prepare competitive applications.

Ms. Chan helps a student pick 3–5 realistic university targets based on their grades, interests, and programme requirements.
1
View current targets
Ranked list showing each target school with real-time match data.

2
Add a new target school
Click "Add School" → modal shows AI-powered recommendations filtered by the student's grades and interests. Each recommendation shows the specific JUPAS programme, fit score, and match percentage. Alternatively, search all 15+ schools manually.
Recommendation example: "City University — Electrical Engineering (JS1221) — 71% fit — 71% match"
After selecting, enter intended major(s) and year of entry. Schools already in the target list are excluded from recommendations.
3
Review eligibility & risk
Each target shows colour-coded signals to guide the counsellor's advice:

Reorder preferences, edit details, track status
Use ↑/↓ to reorder preference ranking. Edit each target to update: intended majors, year of entry, application status (Considering → Applied → Admitted/Rejected/Withdrawn), and student confidence level (slider: Unsure → Decided).
Toggle "By Major" view to see all targets grouped by intended major instead of by school.

Journey 5: Generating an Academic Plan

📄
AI-powered plan generation — personalised advice document
Ms. Chan generates a polished academic plan document that the student can take home.
1
Trigger generation
Click "Generate Plan" from student profile. System collects all student data (grades, IELTS, activities, awards, targets) and starts an async background job.
2
Watch it stream (SSE)
On the Consultant Task page, AI-generated content streams in real-time token-by-token via Server-Sent Events. The counsellor sees the plan being written live. A "Stop Generation" button is available to abort.
/students/{id}/consultant
Plan Output (streaming)
Student Summary
Liu Pui Yi is a Year 5 student at St Paul's College with a best-5 HKDSE aggregate of 22. Her strongest subjects are Chinese Language (5*) and English (5), with solid performance in Mathematics (4). She has expressed interest in…▋
Consultant Chat
Ask about this plan, request changes…
3
Customise the plan
After generation completes, the counsellor can:
Switch Template
Choose Professional (formal), Modern (colourful), or Minimal (clean). Each has distinct colour palette and layout.
Edit Sections
Click "Edit Sections" → select a section (student summary, school rationales, action items) → rich text editor (bold, italic, lists) → Save or Reset to Default.
AI Chat
Type natural language instructions: "Make the summary shorter" or "Add more detail about IELTS preparation." AI returns a JSON patch applied to the plan. Limited to 20 requests per plan per 24 hours.
Export HTML
Download the plan as a standalone HTML file. Prints cleanly via @media print CSS. Counsellor view (with scores/metrics) vs student view (encouraging language, no raw numbers).
Plan contents: Student header • Key metrics (best-5, IELTS, subjects) • Overall assessment • Top 5 recommended schools (with fit score, rationale, SHAP features, risk level) • Action items (task, priority, deadline) • Counsellor-only section (hidden from student print view)

Journey 6: Managing Cohorts

👥
Class-level management — cohorts and bulk operations
Ms. Chan groups students by class for batch operations and analytics.
1
Create cohorts
Navigate to Cohorts → "New Cohort". Enter name (e.g., "5A 2025-26") and description. Typically one cohort per class per academic year.
2
Add students to cohort
Open cohort detail → "Add Students". Search by name, class, or year. Checkbox-select multiple students → "Add Selected". Typically 30 students per class.
3
Bulk edit grades
Click "Bulk Edit Grades" → grid view: rows = students, columns = subjects. Switch between MOCK / TRIAL / OFFICIAL sittings via tab. Edit grades directly in the grid. Ideal for entering mock exam results for an entire class at once.
4
View cohort report
Click "View Report" → cohort analytics showing:
Target Distribution
Which universities are most popular in this class? Bar chart showing student count per target school with average match scores.
Risk Breakdown
How many students have at-risk targets? Helps the counsellor focus attention on students who need intervention.
Subject Performance
Mean grade, variance, and distribution per subject. Filter by sitting (MOCK vs OFFICIAL) to track improvement over time.

Journey 8: School Directory

🏛
Browsing universities — research and explore
Ms. Chan or a student explores the school database to learn about programmes.
1
Search & Filter
Full-text search + filter by type (University/Polytechnic/Community College/Vocational/High School) and location. Shows 15+ seeded Hong Kong institutions. Counsellors can add custom schools.
2
School Profile
Click a school to see: location, programmes, acceptance rates, minimum entry requirements, language requirements. "Add to Targets" button with student picker to quickly assign the school to a student's target list.
3
Export
Export filtered school list as CSV for offline reference or sharing with colleagues.

Journey 9: Graduation & Alumni

🎓
End of cycle — recording outcomes
After JUPAS results, Ms. Chan records where each student was admitted.
1
Mark as Graduated
On student profile, click "Mark as Graduated". Modal asks for: final school (dropdown), final major (text), graduation year (number). Records the actual outcome for institutional analytics.
2
Analytics enrichment
Graduated student data feeds into the Popular Majors analytics and historical trends. Student data is anonymised (SHA256 hash). Profile shows green Graduated 2026 badge.
