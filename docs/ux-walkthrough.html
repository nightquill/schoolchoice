SchoolChoice — UX Walkthrough

How school counsellors actually use the platform, step by step

Hong Kong secondary school context • HKDSE & JUPAS system • Bilingual (EN / 中文) • As of 2026-05-18

Who Uses This

MC
Ms. Chan — School Counsellor
St Paul's College, Hong Kong • Form 5–6 students • HKDSE university guidance
Ms. Chan manages 150 students across 5 classes (5A–5E). She tracks their HKDSE mock/trial/official grades, helps them pick target universities via JUPAS and self-financing pathways, generates personalised academic plans, and monitors cohort-level analytics to spot at-risk students before it’s too late. She uses SchoolChoice daily during the academic year.
LK
Li Ka Ho — Form 6 Student
St Paul's College • Class 6B • Invited by Ms. Chan
Ka Ho received an invite email from his counsellor, set his password, and now logs in with his candidate number. He views his programme choices (Band A–E), checks his academic plan, reviews submission history, and browses the school directory to research backup options.
Context: In Hong Kong, Form 5–6 students sit the HKDSE exam (4 compulsory + 2–4 electives). Universities admit via JUPAS using a “best 5” aggregate score. Each programme has its own formula, weights, and admission statistics. Counsellors help students pick realistic targets and prepare competitive applications. Students who don’t qualify for JUPAS programmes may pursue self-financing sub-degree pathways at Tier 1 institutions (e.g. HKCC, HKU SPACE CC).
Bilingual: The entire UI is available in English and Traditional Chinese (zh-HK). All JUPAS programme names, subject names, self-financing programmes, and search are localised. Users switch language via a toggle in the sidebar. 1,125 translation keys across 13+ components.

3
Bulk Import (CSV/Excel) — Data Only
Import wizard: Upload → Select Sheet (Excel) → Map Columns → Review & Commit. Import creates student data records only — no user accounts are auto-created. Students appear as “unaccounted” until invited.
Import hardening: Chinese header support (姓名, 英文, 數學 → auto-mapped) • Grade normalisation (percentages, letter grades → HKDSE scale) • Encoding detection (Big5, GB2312 via chardet) • Row merging for duplicate candidate numbers • Profile-only and partial data import • File validation (rejects PDFs, corrupt files, >2,000 rows) • Requires data_import permission on the cohort.
4
Create Teacher Groups & Set Permissions

Per-group, per-cohort permissions: Visibility toggle + 8 tool access levels (None / View / Edit).
Tools: Programme Choices • Grades • Plans • Submissions • Reports • Cohort Mgmt • Data Import • Account Assignment.
When a teacher belongs to multiple groups, the most permissive level wins.

Journey 5: Generating an Academic Plan

📄
AI-powered plan generation — personalised advice document
Ms. Chan generates a polished academic plan document that the student can take home.
1
Trigger generation
Click “Generate Plan” from student profile. System collects all student data (grades, IELTS, activities, awards, targets) and starts an async background job.
2
Watch it stream (SSE)
On the Consultant Task page, AI-generated content streams in real-time token-by-token via Server-Sent Events. The counsellor sees the plan being written live. A “Stop Generation” button is available to abort.
/students/{id}/consultant
Plan Output (streaming)
Student Summary

Consultant Chat
Ask about this plan, request changes…
3
Customise the plan
After generation completes, the counsellor can:
Switch Template
Choose Professional (formal), Modern (colourful), or Minimal (clean). Each has distinct colour palette and layout.
Edit Sections
Click “Edit Sections” → select a section (student summary, school rationales, action items) → rich text editor (bold, italic, lists) → Save or Reset to Default.
AI Chat
Type natural language instructions: “Make the summary shorter” or “Add more detail about IELTS preparation.” AI returns a JSON patch applied to the plan. Limited to 20 requests per plan per 24 hours.
Export HTML & PDF
Download the plan as a standalone HTML file or PDF (generated via PyMuPDF). Prints cleanly via @media print CSS. Counsellor view (with scores/metrics) vs student view (encouraging language, no raw numbers).
Plan contents: Student header • Key metrics (best-5, IELTS, subjects) • Overall assessment • Top 5 recommended schools (with fit score, rationale, SHAP features, risk level) • Action items (task, priority, deadline) • Counsellor-only section (hidden from student print view)

Journey 6: Managing Cohorts

👥
Class-level management — cohorts and bulk operations
Ms. Chan groups students by class for batch operations and analytics.
1
Create cohorts
Navigate to Cohorts → “New Cohort”. Enter name (e.g., “5A 2025-26”) and description. Typically one cohort per class per academic year.
2
Add students to cohort
Open cohort detail → “Add Students”. Search by name, class, or year. Checkbox-select multiple students → “Add Selected”. Typically 30 students per class.
3
Bulk edit grades
Click “Bulk Edit Grades” → grid view: rows = students, columns = subjects. Switch between MOCK / TRIAL / OFFICIAL sittings via tab. Edit grades directly in the grid. Ideal for entering mock exam results for an entire class at once.
4
View cohort report
Click “View Report” → cohort analytics showing:
Target Distribution
Which universities are most popular in this class? Bar chart showing student count per target school with average match scores.
Risk Breakdown
How many students have at-risk targets? Helps the counsellor focus attention on students who need intervention.
Subject Performance
Mean grade, variance, and distribution per subject. Filter by sitting (MOCK vs OFFICIAL) to track improvement over time.

Journey 8: School Directory

🏛
Browsing universities — JUPAS and self-financing pathways
Ms. Chan or a student explores the school database to learn about programmes across both JUPAS and self-financing institutions.
1
Search & Filter
Full-text search (English and Chinese) + filter by type (University/Polytechnic/Community College/Vocational/High School) and location. Shows 15+ seeded Hong Kong institutions. Counsellors can add custom schools.
2

Click a school to see: location, programmes, acceptance rates, minimum entry requirements, language requirements. “Add to Targets” button with student picker to quickly assign the school to a student’s target list.

Each SF programme includes admission score data (best-5 HKDSE aggregate). Students are scored against SF programmes the same way as JUPAS — match percentage and eligibility badges apply.
4
Export
Export filtered school list as CSV for offline reference or sharing with colleagues.

Journey 10: Student Onboarding & Accounts

👤
Invite-based student accounts — from import to first login
Students don’t self-register. The counsellor imports data, then explicitly invites students to create their own accounts.
1
Import CSV creates data only
When Ms. Chan imports a student roster, the system creates student data records but no user accounts. All imported students appear as No Account (unaccounted) in the student list.
2
Teacher enters student email
A teacher with account_assignment permission on the cohort opens a student’s profile and enters their email address in the Account section. Clicks “Send Invite”.
3
Signed invite link sent via email
The system generates a signed, time-limited invite URL and emails it to the student. Student status changes to Invited.
4
Student opens link & sets password
Student clicks the invite link in their email, is taken to a registration page pre-filled with their details. They set a password. Account is auto-created and linked to the existing student data record.
5
Student logs in
Student can now log in via candidate number or email. Status changes to Active. They see their own student portal with programme choices, plans, submissions, and school directory.
6
Password reset
Standard password reset flow via email for students who forget their credentials.
Unaccounted filter: On the dashboard, the “Show Unaccounted” toggle isolates students imported via CSV who have not yet been invited. This helps the counsellor batch-invite remaining students efficiently.

Journey 11: Teacher Group Permissions (Admin)

🔒
Role-based access — who sees what
The admin creates teacher groups, assigns members, and configures per-cohort tool access so each teacher sees only what they need.
1
Create a teacher group
Admin navigates to /admin/teacher-groups → “New Group”. Enters group name (e.g. “Form 5 Teachers”, “Science Dept”).
2
Add teachers to the group
Select from registered teacher accounts and assign them to the group. A teacher can belong to multiple groups.
3
Set per-cohort permissions
For each cohort, configure:
Visibility toggle: Can the group see this cohort at all?
8 tool access levels (None / View / Edit):
ToolNoneViewEdit
Programme ChoicesHiddenRead-only listAdd/remove/reorder targets
GradesHiddenSee grade tableEnter/modify grades
PlansHiddenRead generated plansGenerate & edit plans
SubmissionsHiddenView submission historyAccept/reject submissions
ReportsHiddenView cohort reportsExport reports
Cohort MgmtHiddenSee member listAdd/remove students
Data ImportHidden—Import CSV/Excel

4
Most permissive wins
When a teacher belongs to multiple groups, the system applies the most permissive access level across all their groups for each tool on each cohort. The teacher sees only their allowed cohorts and the tools they have access to.

Student login
Student logs in via candidate number or email at /login. Redirected to the student dashboard.
2

The student’s home screen shows their programme choices organised by JUPAS band (A–E). Each choice shows the programme name, institution, match percentage, and eligibility status. Students can see their counsellor’s recommendations but cannot modify the list.
