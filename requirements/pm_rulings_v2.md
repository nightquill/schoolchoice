## Purpose

This file records PM rulings issued during the v2 pipeline run. A ruling is issued when:
- An implementation agent raises a conflict between requirements
- A requirement is ambiguous and needs authoritative interpretation
- An agent proposes a deviation from preferences.md and PM approval is required
- A schema or API change affects multiple agents and must be coordinated

Rulings are append-only. No ruling may be deleted.

Format:
```
## RULING-v2-NNN
- Date: YYYY-MM-DD
- Raised by: <agent>
- Issue: <description>
- Decision: <approved/rejected/deferred>
- Rationale: <explanation>
- Affected REQ-IDs: <list>
```
